NDL Website First Use Tutorial

This tutorial starts from a blank Windows computer and shows how to download the NDL website project, make simple updates, preview them, and publish them to GitHub Pages.

For regular use: after setup is complete, most updates can be done by double-clicking RUN_SITE_TOOL.bat.

What You Are Working With

NameSimple meaning
Website repositoryThe folder containing all files for the lab website.
GitThe software that downloads the website project and saves/publishes changes.
GitHubThe online place where the official copy of the website files is stored.
GitHub PagesThe service that turns the repository into the public website.
JekyllThe website builder used by GitHub Pages.
RUN_SITE_TOOL.batA double-click menu for common editing tasks.

Part 1: Install The Required Software

1. Install Git For Windows

  1. Open https://git-scm.com/download/win.
  2. Download and run the Windows installer.
  3. For most screens, keep the default choices and click Next.
  4. Make sure Git Bash is installed. The website tool uses it.

2. Install Python 3

  1. Open https://www.python.org/downloads/windows/.
  2. Download and run Python 3 for Windows.
  3. Important: tick Add python.exe to PATH if the installer shows that option.
  4. Click Install Now.

3. Optional: Install Ruby For Local Preview

GitHub Pages can build the website online after publishing. Ruby is useful if you want to preview or test the site before publishing.

  1. Open https://rubyinstaller.org/downloads/.
  2. Download the recommended Ruby+Devkit installer.
  3. Run the installer and keep the default choices.
  4. After installation, open Git Bash in the website folder and run bundle install once.

Part 2: Download The Website Project

1. Choose A Folder

Choose where the website folder should live. For example:

Documents\Neuromorphic Device Lab\Repository

2. Open Git Bash In That Folder

  1. Open the parent folder in File Explorer.
  2. Right-click empty space inside the folder.
  3. Choose Open Git Bash here.

3. Clone The Repository

In Git Bash, type or paste:

git clone https://github.com/ndl-ntu/ndl-ntu.github.io.git

This creates a new folder named ndl-ntu.github.io.

Part 3: Before Editing, Pull The Latest Version

If another person may have edited the website, always pull first.

Pull With Git Bash

  1. Open the ndl-ntu.github.io folder.
  2. Right-click empty space and choose Open Git Bash here.
  3. Run:
git pull origin master

Part 4: Use The Click-To-Run Tool

Open The Menu

  1. Double-click RUN_SITE_TOOL.bat.
  2. A terminal window opens.
  3. Choose a number and press Enter.
1. Write a news post
2. Add research staff/student/alumni
3. Remove research staff/student/alumni
4. Add publication by DOI
5. Run Jekyll build
6. Publish changes to GitHub
0. Exit

Common Examples

  • Choose 1 to add news. The post appears on the News page.
  • Choose 2 to add a staff member, student, intern, or alumni profile.
  • Choose 3 to remove a person profile by name.
  • Choose 4 to add a publication by DOI. The tool fetches Crossref metadata and adds the citation badge.
  • Choose 5 to check that Jekyll can build the site.

Part 5: Preview The Website

Simple Build Check

In the tool menu, choose 5. Run Jekyll build. If it finishes without an error, the source files are valid.

Browser Preview

If Ruby/Jekyll is installed, open Git Bash in the website folder and run:

bundle exec jekyll serve

Then open the address shown in the terminal, usually http://127.0.0.1:4000/.

Part 6: Publish Changes To GitHub

Yes, publishing is integrated in the BAT menu. Choose 6. Publish changes to GitHub. The tool will show changed files, optionally run a build, ask for a commit message, commit all changed files, and push to GitHub.

Publish With The Menu

  1. Double-click RUN_SITE_TOOL.bat.
  2. Choose 6.
  3. Review the changed files shown by the tool.
  4. Choose whether to run Jekyll build.
  5. Enter a short commit message, for example Update people and publications.
  6. Confirm that you want to commit and push.

After pushing, GitHub Pages may take a few minutes to update the public website.

Part 7: If Something Goes Wrong

ProblemWhat to try
bash was not foundInstall Git for Windows, then double-click RUN_SITE_TOOL.bat again.
python was not foundInstall Python 3 and tick Add python.exe to PATH.
Git asks for loginSign in with the GitHub account that has permission to edit the repository.
Build error after editing peopleThere may be a spacing problem in _data/people.yml. Compare the changed entry with nearby entries.
Live website does not update immediatelyWait a few minutes. GitHub Pages needs time to rebuild.
