Neuromorphic Device Lab Website Editing Guide

This guide explains how the NDL website repository is organized and which files control each visible part of the website. It is written for maintainers who may not have programming, Git, HTML, or Markdown experience.

Most routine edits should use the click-to-run tool: double-click RUN_SITE_TOOL.bat in the repository folder. For first-time setup, start with the first use tutorial.

Basic Ideas

The website is a GitHub Pages site built with Jekyll. Jekyll reads text files in this folder and turns them into website pages.

TermPlain meaning
Repository, or repoThe folder that contains all website files and the saved history of changes.
GitThe history system. It records what changed, who changed it, and allows the site to be pushed to GitHub.
GitHub PagesThe hosting service that publishes this repository as the public website.
JekyllThe website builder. It converts Markdown, YAML, layouts, and styles into HTML pages.
MarkdownA simple text format. For example, ## Title becomes a heading and * item becomes a bullet.
HTMLThe web page language. It is used when Markdown is not flexible enough, for example publication badges or profile sections.
YAMLA structured text format used for lists such as people and navigation. Spaces are important in YAML.

Routine Edits

Use The Click-To-Run Tool

  1. Open the repository folder.
  2. Double-click RUN_SITE_TOOL.bat.
  3. Choose one action from the menu.
  4. Follow the questions shown in the terminal window.
  5. Preview the website before publishing.

The tool can create a news post, add or remove people, add a publication by DOI, run a Jekyll build, and publish changes to GitHub.

Manual Editing Rules

  • Use a plain text editor such as VS Code, Notepad++, or GitHub web editor.
  • Do not edit files in _site/. That folder is generated output and should not be used as the source.
  • Keep filenames simple: English letters, numbers, hyphens, and dots are safest.
  • After changing a file, preview locally or let GitHub Pages build it.

Page By Page Guide

Homepage: index.md

The first part between --- lines is page information for Jekyll. Do not remove it. The visible homepage content starts after the second ---.

Text in fileWhere it appears
<h1>Neuromorphic Device Lab</h1>Main lab name at the top of the homepage.
lab-mission paragraphOne-sentence homepage mission.
lab-theme-row linksThe three compact research area buttons.
{% include notification.html ... %}The dismissible recruitment notice on the homepage.
## Research ThemesThe three homepage research blocks.
## Selected Recent PublicationsShort selected publication list on the homepage.
## Principal InvestigatorPI biography lower on the homepage.

Research Page: highlights.md

This is the research profile page. Each research theme is written as a block with an image, a title, a short summary, related papers, and active people. Images used here normally live in assets/img/.

Join Us Page: join-us.md

This should stay as a standing recruitment page. Update it when research topics, eligibility, scholarship routes, or email instructions change. News posts can link to this page instead of repeating all recruitment information.

People Page

The People page has two source files:

  • people.md controls the page title and the FYP student list.
  • _data/people.yml controls profile cards for PI, research staff, graduate students, interns, and alumni.

In _data/people.yml, each section starts with a title such as:

- title: Graduate Research Student
  images:
    - link: /assets/img/blank.png
      alt: Mr. Example Student
      description: |-
        ### Mr. Example Student
        #### About
        Mr. Example Student is now a PhD student at NTU EEE.
      ratio: is-16by9
FieldMeaning
titleThe group heading, for example Research Staff or Alumni.
linkThe image shown for the person. Use /assets/img/name.jpg for local images or a full web URL.
large_linkOptional larger image used when opening the image. Often same as link.
altPerson name used for accessibility and image fallback.
descriptionThe text inside the profile card. Markdown headings and links work here.
ratioImage shape, such as is-1by1, is-3by4, is-4by3, or is-16by9.
YAML spacing matters. Keep the same indentation as nearby people. If spacing is changed incorrectly, Jekyll may fail to build.

Publications

The publications page is publications.md. It has a top navigation area, a Selected Publications section, a Recent Publications section, and year headings.

A typical publication line looks like this:

* Author A, Author B, and D. S. Ang. Paper title. **Journal Name**. 2026. [doi: 10.xxxx/example](https://doi.org/10.xxxx/example){:target="_blank"} <span class="publication-metrics"><span class="__dimensions_badge_embed__" data-doi="10.xxxx/example" data-style="small_rectangle" data-hide-zero-citations="true"></span></span>
PartWhere it appears
*Starts one bullet item.
**Journal Name**Shows the journal in bold.
[doi: ...](https://doi.org/...)Clickable DOI link.
publication-metricsInline citation badge beside the publication.
tag is-warning is-lightSmall "Highly Cited" label in Selected Publications.

For routine publication updates, use RUN_SITE_TOOL.bat and choose Add publication by DOI. The tool inserts metadata and the citation widget automatically.

News Posts

News posts are files in _posts/. The filename must start with a date:

2026-06-14-example-news-title.md

Each post starts with a small header:

---
title: Example news title
layout: post
author: NDL
---

Write the news here.

The post appears automatically on the News page at /blog/. The date comes from the filename.

Images

Put local images in assets/img/. Recommended formats are .jpg, .png, and .svg. Keep image files reasonably small for fast loading.

Preview And Publish

Preview Locally

If Ruby/Jekyll is installed, run this in the repository folder:

bundle exec jekyll serve

Then open the local address shown by the command, usually http://127.0.0.1:4000/. For a build-only check, run:

bundle exec jekyll build

Publish To GitHub Pages

Publishing requires Git. A typical command sequence is:

git status
git add .
git commit -m "Describe the website update"
git push origin master

After pushing, GitHub Pages rebuilds the public website. It may take a few minutes before the live site updates.

Safety Checklist

  • Before editing, make sure you are in the repository folder, not the generated _site folder.
  • For people edits, keep YAML indentation exactly like nearby entries.
  • For publication edits, keep DOI text and citation widget DOI the same.
  • After image changes, confirm the image path starts with /assets/img/ or a full https:// URL.
  • Run bundle exec jekyll build before publishing when possible.
  • Use a clear Git commit message, for example Update people and publications.
